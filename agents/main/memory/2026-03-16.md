### 系统架构理解
**两个层级的结构：**
- `agents/*/` - 存储代理的运行时数据（会话、配置、认证）
- `workspace-*/` - 存储代理的工作数据（项目、文件、记忆）

**多代理配置（来自 openclaw.json）：**
```
main:   agents/main/agent ←→ workspace/
writer: agents/writer/agent ←→ workspace-writer/
coder:  agents/coder/agent ←→ workspace-coder/
test:   agents/test/agent ←→ workspace-test/
```

## 关键认识

1. **多代理架构的必要性**
   - 每个代理都需要独立的 SOUL.md（人设）、USER.md（用户信息）等
   - 这不是重复，而是设计

2. **验收的重要性**
   - Agent 的口头承诺不可信
   - 必须实际验证任务完成情况

3. **系统整洁性**
   - 需要定期检查和清理多余的目录
   - 避免配置和实际文件的不一致
